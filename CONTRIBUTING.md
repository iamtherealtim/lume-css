# Contributing to Lume CSS

Thanks for your interest in improving Lume.

## Development Guidelines

- Keep changes minimal and focused.
- Match existing naming and style conventions.
- Avoid introducing breaking class-name changes without discussion.
- Ensure examples in docs and showcase still render correctly.

## Local Workflow

1. Fork and clone the repository.
2. Create a feature branch.
3. Make your changes.
4. Validate pages:
   - `docs/lume-showcase.html`
   - `docs/lume-docs.html`
   - `docs/admin-demo/`
5. Open a pull request with a clear summary and screenshots for UI changes.

## Pull Request Checklist

- [ ] Change is scoped and intentional
- [ ] No broken links in docs/showcase
- [ ] Light + dark mode both checked
- [ ] Keyboard/focus states checked for interactive elements
- [ ] Changelog updated when user-visible behavior changes
